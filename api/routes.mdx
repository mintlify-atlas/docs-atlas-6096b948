## Overview

KYA Hukuk & Danışmanlık Flask application exposes two main categories of routes:
- **Public routes**: Accessible to all visitors
- **Admin routes**: Protected by Flask-Login authentication

### Contact form submission

<ParamField path="/iletisim/gonder" method="POST">
  Handles contact form submissions with server-side validation
  
  **Auth required**: No (CSRF protection enabled)
  
  **Form parameters**:
  - `name` (string, required): Sender name (min 2 characters)
  - `email` (string, required): Valid email address
  - `subject` (string, optional): Message subject
  - `message` (string, required): Message content (min 10 characters)
  
  **Validation rules**:
  - Name must be at least 2 characters
  - Email must match regex: `^[^@\s]+@[^@\s]+\.[^@\s]+$`
  - Message must be at least 10 characters
  
  **Success**: Saves message to database, logs to app logger, redirects to `/iletisim` with success flash
  
  **Error**: Redirects to `/iletisim` with error flash messages
  
  **Database operations**:
  - Creates new `ContactMessage` record with IP address
</ParamField>

## Admin routes

All admin routes require authentication via Flask-Login. Unauthenticated users are redirected to `/admin/login`.

### Admin login

<ParamField path="/admin/login" method="GET">
  Displays admin login form
  
  **Auth required**: No
  
  **Returns**: HTML template `admin/login.html`
  
  **Behavior**: Redirects to dashboard if already authenticated
</ParamField>

<ParamField path="/admin/login" method="POST">
  Authenticates admin user
  
  **Auth required**: No
  
  **Form parameters**:
  - `username` (string): Admin username
  - `password` (string): Admin password
  
  **Success**: 
  - Logs in user with `login_user()`
  - Sets `session.permanent = True` (3-hour session lifetime)
  - Redirects to `next` parameter or dashboard
  
  **Error**: Redirects to login page with error flash
  
  **Security checks**:
  - User exists in database
  - Password hash matches
  - User is active (`is_active=True`)
</ParamField>

### Admin logout

<ParamField path="/admin/logout" method="GET">
  Logs out the current admin user
  
  **Auth required**: Yes
  
  **Returns**: Redirects to `/admin/login`
</ParamField>

### Admin dashboard

<ParamField path="/admin" method="GET">
  Main admin dashboard with statistics
  
  **Auth required**: Yes
  
  **Aliases**: `/admin/dashboard`
  
  **Returns**: HTML template `admin/dashboard.html`
  
  **Statistics provided**:
  - Active team members count
  - Published articles count
  - Draft articles count
  - Active practice areas count
  - Total contact messages
  - Unread contact messages
  
  **Additional data**:
  - 5 most recent articles (by `created_at`)
</ParamField>

### Create team member

<ParamField path="/admin/ekip/yeni" method="GET">
  Displays form to add a new team member
  
  **Auth required**: Yes
  
  **Returns**: HTML template `admin/team_form.html` with `member=None`
</ParamField>

<ParamField path="/admin/ekip/yeni" method="POST">
  Creates a new team member
  
  **Auth required**: Yes
  
  **Form parameters**:
  - `name` (string, required): Team member name
  - `role` (string): Position/role
  - `bio` (text): Biography
  - `linkedin_url` (string): LinkedIn profile URL
  - `photo` (file): Photo upload
  - `photo_url_ext` (string): External photo URL (used if no file uploaded)
  - `order_index` (integer): Display order (default: 0)
  - `is_active` (checkbox): Active status
  
  **File handling**:
  - Uploads photo to GitHub if `GITHUB_TOKEN` configured
  - Falls back to local storage in development
  - Vercel environments require GitHub token
  
  **Success**: Creates member, redirects to `/admin/ekip` with success flash
</ParamField>

### Edit team member

<ParamField path="/admin/ekip/<int:mid>/duzenle" method="GET">
  Displays form to edit existing team member

</ParamField>

<ParamField path="/admin/ekip/<int:mid>/duzenle" method="POST">
  Updates an existing team member
  
  **Auth required**: Yes
  
  **URL parameters**:
  - `mid` (integer): Team member ID
  
  **Form parameters**: Same as create endpoint
  
  **Photo handling**:
  - If new photo uploaded: deletes old photo from GitHub, uploads new one
  - If external URL provided: deletes old photo, uses external URL
  - Otherwise: keeps existing photo
  
  **Success**: Updates member, redirects to `/admin/ekip` with success flash
</ParamField>

### Delete team member

  - Redirects to `/admin/ekip` with info flash
</ParamField>

</ParamField>

### Create article

<ParamField path="/admin/makaleler/yeni" method="GET">
  Displays form to create a new article
  
  **Auth required**: Yes
  
  **Returns**: HTML template `admin/article_form.html` with `article=None`
</ParamField>

<ParamField path="/admin/makaleler/yeni" method="POST">
  Creates a new article
  
  **Auth required**: Yes
  
  **Form parameters**:
  - `title` (string, required): Article title
  - `slug` (string, optional): Custom URL slug (auto-generated from title if empty)
  - `summary` (text): Article summary/excerpt
  - `content` (text): Full article content (HTML)
  - `author` (string): Author name
  - `cover` (file): Cover image upload
  - `cover_url_ext` (string): External cover image URL
  - `is_published` (checkbox): Publish immediately
  
  **Slug generation**:
  - Auto-slugifies title if custom slug not provided
  - Ensures uniqueness by appending counter if slug exists
  - Turkish character normalization (ş→s, ğ→g, etc.)
  
  **Publishing**:
  - If `is_published` checked: sets `published_at` to current UTC time
  - Otherwise: saves as draft with `published_at=None`
  
  **Success**: Creates article, redirects to `/admin/makaleler` with success flash
</ParamField>

### Edit article

<ParamField path="/admin/makaleler/<int:aid>/duzenle" method="GET">
  Displays form to edit existing article
  
  **Auth required**: Yes
  
  **URL parameters**:
  - `aid` (integer): Article ID
  
  **Returns**: HTML template `admin/article_form.html` or 404 if not found
</ParamField>

<ParamField path="/admin/makaleler/<int:aid>/duzenle" method="POST">
  Updates an existing article
  
  **Auth required**: Yes
  
  **URL parameters**:
  - `aid` (integer): Article ID
  
  **Form parameters**: Same as create endpoint
  
  **Slug update logic**:
  - Only updates slug if custom slug provided AND different from current
  - Ensures new slug doesn't conflict with other articles
  
  **Publishing logic**:
  - If article was draft and now published: sets `published_at` to current time
  - If already published: keeps original `published_at`
  
  **Cover image handling**:
  - Same deletion/replacement logic as team photos
  
  **Success**: Updates article, redirects to `/admin/makaleler` with success flash
</ParamField>

### Delete article

</ParamField>

### Save practice areas (bulk)

<ParamField path="/admin/alanlar/kaydet" method="POST">
  Bulk saves all practice areas (delete all + re-insert)
  
  **Auth required**: Yes
  
  **Form parameters** (arrays):
  - `title[]` (string[]): Area titles
  - `desc[]` (string[]): Area descriptions
  - `icon[]` (string[]): Font Awesome icon classes
  - `image_url[]` (string[]): Existing/external image URLs
  - `image_file[]` (file[]): New image uploads
  - `active[]` (string[]): Indices of active areas
  
  **Processing logic**:
  - Deletes all existing practice areas
  - Iterates through arrays by index
  - For each area:
    - Uses uploaded file if provided, otherwise keeps existing URL
    - Sets `order_index` based on array position
    - Marks as active if index is in `active[]` array
  - Skips empty titles
  
  **Success**: Recreates all areas, redirects to `/admin/alanlar` with success flash
</ParamField>

## Middleware and hooks

### Admin subdomain redirect

**Decorator**: `@app.before_request`

**Purpose**: Redirects `admin.*` subdomains to main domain `/admin/*` paths

**Behavior**:
- Checks `Host` header for `admin.` prefix
- Performs 301 redirect to `https://www.kyahukukdanismanlik.site{request.full_path}`

### Context processor

**Decorator**: `@app.context_processor`

**Purpose**: Injects global variables into all templates

**Variables injected**:
- `settings`: Site settings dictionary (cached 60s)
- `unread_messages_count`: Count of unread contact messages (admin pages only)

### User loader

**Decorator**: `@login_manager.user_loader`

**Purpose**: Flask-Login callback to reload user from session

**Implementation**: `app.py:33-35`

```python
@login_manager.user_loader
def load_user(user_id):
    return db.session.get(User, int(user_id))
```
