## Overview

KYA Hukuk & Danışmanlık uses SQLAlchemy ORM with support for both PostgreSQL (production) and SQLite (development). All models are defined in `models.py` and use `datetime.utcnow` for timestamps.

## User

**Table name**: `user`

**Purpose**: Admin authentication and authorization

**Inherits**: `UserMixin`, `db.Model`

### Fields

<ParamField path="id" type="Integer" required>
  Primary key
</ParamField>

<ParamField path="username" type="String(80)" required>
  Unique username for login
  
  **Constraints**: Unique, not nullable
</ParamField>

<ParamField path="email" type="String(120)">
  User email address (optional)
  
  **Constraints**: Unique, nullable
</ParamField>

<ParamField path="password_hash" type="String(256)" required>
  Hashed password using Werkzeug's `generate_password_hash`
  
  **Security**: Never stores plain-text passwords
</ParamField>

<ParamField path="is_active" type="Boolean" default="True">
  Whether user can log in
  
  **Default**: True
</ParamField>

<ParamField path="created_at" type="DateTime" default="utcnow">
  Account creation timestamp
  
  **Default**: `datetime.utcnow`
</ParamField>

### Methods

<ResponseField name="set_password(password)">
  Hashes and stores password
  
  **Parameters**:
  - `password` (string): Plain-text password
  
  **Implementation**: Uses Werkzeug's `generate_password_hash`
  
  **Source**: `models.py:18-19`
</ResponseField>

<ResponseField name="check_password(password)">
  Verifies password against stored hash
  
  **Parameters**:
  - `password` (string): Plain-text password to verify
  
  **Returns**: Boolean
  
  **Implementation**: Uses Werkzeug's `check_password_hash`
  
  **Source**: `models.py:21-22`
</ResponseField>

### Example usage

```python
# Create user
user = User(username='admin', is_active=True)
user.set_password('secure_password')
db.session.add(user)
db.session.commit()

# Verify password
if user.check_password('password_attempt'):
    login_user(user)
```

- `contact_phone`: Phone number

- `contact_hours`: Business hours

- `about_short`: Short description
- `logo_url`: Color logo for light backgrounds
- `logo_white_url`: White logo for dark backgrounds
- `footer_text`: Copyright text

## Database configuration

### Connection setup

**Source**: `config.py`

<Tabs>
  <Tab title="PostgreSQL (Production)">
    **URL sources** (priority order):
    1. `DATABASE_URL`
    2. `DATABASE_URL_UNPOOLED` (Neon serverless)
    3. `POSTGRES_URL`
    4. `POSTGRES_URL_NO_SSL`
    
    **Driver**: `postgresql+psycopg2`
    
    **Connection pooling**:
    ```python
    SQLALCHEMY_ENGINE_OPTIONS = {
        'pool_size': 1,           # Single connection for serverless
        'max_overflow': 2,        # Allow 2 additional connections
        'pool_timeout': 10,       # 10s connection timeout
        'pool_recycle': 300,      # Recycle connections every 5 min
        'pool_pre_ping': True,    # Detect stale connections
    }
    ```
    
    **SSL**: Enabled via `sslmode=require` in connection string
  </Tab>
  
  <Tab title="SQLite (Development)">
    **Path**: 
    - Vercel: `/tmp/kya.db` (ephemeral)
    - Local: `{BASE_DIR}/database/kya.db`
    
    **Connection string**: `sqlite:///{path}`
    
    **Pooling**: Disabled (SQLite default)
    
    **Note**: Vercel SQLite is temporary and resets between deployments. Production must use PostgreSQL.
  </Tab>
</Tabs>

### Initialization

**Source**: `app.py:735-928`

The `init_db()` function:

1. Creates all tables with `db.create_all()`
2. Syncs admin users from environment variables
3. Seeds default settings (contact info, logos, SEO)
4. Seeds default hero sections for all pages
5. Creates sample team members if table empty
6. Creates sample practice areas if table empty
7. Creates sample articles if table empty
8. Runs migrations (e.g., adds `image_url` to practice areas)

**Automatic execution**: Runs on app startup (line 974)

**Manual trigger**: `/setup?key={SETUP_KEY}` endpoint

### Migrations

**Pattern**: Inline migrations in `init_db()`

**Example** (`app.py:808-819`):
```python
from sqlalchemy import text, inspect

inspector = inspect(db.engine)
cols = [c['name'] for c in inspector.get_columns('practice_area')]
if 'image_url' not in cols:
    with db.engine.connect() as conn:
        conn.execute(text('ALTER TABLE practice_area ADD COLUMN image_url VARCHAR(500)'))
        conn.commit()
```

## Relationships

### Current schema

The application uses a **flat schema** with no foreign key relationships between models:

- `User` ↔ No relationships (isolated authentication)
- `Article` ↔ No author FK (stores author name as string)
- `TeamMember` ↔ No relationships (standalone)
- `PracticeArea` ↔ No relationships (standalone)
- `ContactMessage` ↔ No relationships (standalone)
- `HeroSection` ↔ No relationships (one-to-one with page names)
- `SiteSetting` ↔ No relationships (key-value store)
