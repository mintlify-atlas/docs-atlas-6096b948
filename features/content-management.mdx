Manage hero (banner) sections for all pages at `/admin/hero`. Each page has a dedicated hero with title, subtitle, and background image.

## File storage

The system supports two storage modes:

<Tabs>
  <Tab title="GitHub (production)">
    When `GITHUB_TOKEN` is set, files are uploaded to the GitHub repository:
    
    1. Files are saved to `static/uploads/{filename}` in the repo
    2. Returns raw GitHub URL: `https://raw.githubusercontent.com/{owner}/{repo}/main/static/uploads/{filename}`
    3. Implements file overwrite detection using SHA comparison
    
    ```python
    # From app.py:68
    def _upload_to_github(filename, file_bytes):
        path = f'static/uploads/{filename}'
        api_url = f'{_GH_API}/{path}'
        # Check existing file for SHA
        existing = http_client.get(api_url, headers=headers, timeout=10)
        sha = existing.json().get('sha') if existing.status_code == 200 else None
    ```
  </Tab>
  
  <Tab title="Local (development)">
    Without `GITHUB_TOKEN`, files are stored in the local `static/uploads/` directory.
    
    <Warning>
      Vercel's filesystem is read-only. Local storage will fail in production without GitHub integration.
    </Warning>
  </Tab>
</Tabs>

## Settings cache

For performance, site settings are cached with a 60-second TTL:

```python
# From app.py:209
def site_settings() -> dict:
    global _settings_cache, _settings_cache_ts
    if time.time() - _settings_cache_ts < _SETTINGS_TTL and _settings_cache:
        return _settings_cache
    _settings_cache = {s.key: s.value for s in SiteSetting.query.all()}
    _settings_cache_ts = time.time()
    return _settings_cache
```

The cache is invalidated when you save settings via `_invalidate_settings_cache()` in app.py:224.

<Note>
  All templates access settings via the `settings` context variable, injected by the `inject_settings()` processor in app.py:230.
</Note>
